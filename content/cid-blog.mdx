Après avoir développé plusieurs interfaces frontend avec React, j'ai souhaité mieux comprendre la partie Backend des applications web.

Laravel s'est rapidement imposé comme un framework intéressant grâce à sa structure claire et ses nombreux outils.

CID Blog est un projet qui m'a permis d'explorer cet environnement.

<Callout>Pourquoi Laravel ?</Callout>

Laravel offre une approche organisée pour construire des applications web.

À travers ce projet, j'ai découvert :

* la structure MVC ;

* la gestion des routes ;

* les modèles et migrations ;

* l'interaction avec une base de données ;

* la création de fonctionnalités dynamiques.

<Callout>L'utilisation de Livewire</Callout>

Livewire m'a permis de créer des interactions dynamiques tout en restant dans l'écosystème Laravel.

Cette approche m'a aidé à comprendre comment créer des interfaces réactives sans dépendre uniquement d'un framework frontend.

<Callout>Ce que ce projet m'a apporté</Callout>

CID Blog a renforcé mon intérêt pour le développement Backend.

Il m'a permis de mieux comprendre le fonctionnement interne d'une application web complète et d'avoir une vision plus globale du développement.
