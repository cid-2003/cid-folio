Au début de mon apprentissage du développement web, JavaScript était mon principal outil pour créer des interfaces interactives avec React.

Cette approche m'a permis de comprendre les bases du développement frontend : composants, états, appels API et gestion des interactions utilisateur.

Cependant, en développant des projets plus importants, j'ai rapidement rencontré les limites d'un code uniquement basé sur JavaScript.

C'est à ce moment que j'ai commencé à utiliser TypeScript.

<Callout>Pourquoi adopter TypeScript ?</Callout>

Avec l'évolution d'un projet, le code devient naturellement plus complexe.

TypeScript m'a permis d'obtenir :

* une meilleure compréhension des données manipulées ;
 
* moins d'erreurs pendant le développement ;

* une meilleure autocomplétion ;

* un code plus facile à maintenir.

<Callout>Mon expérience avec mes projets</Callout>

J'ai utilisé TypeScript notamment sur :

* CID Movie, une application React utilisant l'API TMDB ;

* Arbre du savoir, une application de recherche de livres utilisant Google Books API ;

* SikaJob, une application Full Stack développée avec Next.js.

Ces projets m'ont permis de comprendre l'intérêt du typage dans des applications qui évoluent.

<Callout>Ce que j'ai appris</Callout>

TypeScript n'est pas seulement une technologie supplémentaire à ajouter à un projet.

C'est une manière différente de réfléchir à son code, en anticipant davantage les erreurs et en construisant des applications plus solides.

Aujourd'hui, TypeScript fait partie des outils que je privilégie pour mes nouveaux projets.
